Run a check on this page:

  • Reports the computed accessible name of every interactive element (per W3C AccName 1.2). Flags elements with no name, and notes when the name comes from a placeholder only — not a spec accname.

  • WCAG 2.2.2, 2.3.1, 2.3.3. Inventories every CSS animation and matches each to its @keyframes body. Flags infinite iteration (spinner-heuristic downgrade), duration > 5s, problematic keyframe patterns (flash, shake, fast rotate), missing @media (prefers-reduced-motion), <marquee>/<blink>, and animated-GIF candidates.

  • Validates every role and aria-* attribute. Flags unknown roles, abstract roles, unknown aria-* names (with "did you mean" suggestions), broken aria-labelledby/describedby/controls/owns ID refs, invalid boolean values, missing required attributes per role, and role="presentation"/"none" on focusable elements.

  • WCAG 2.1.1, 4.1.2, 3.3.1/3.3.2. Inventories every <button>, <a>, interactive-role element, and non-interactive element with inline handlers. Flags <button> without explicit type= inside a form, <a href="#"> used as buttons, <a> without href, div/span onclick without role/tabindex, and inconsistent aria-pressed/expanded/haspopup states.

  • Text vs background contrast against WCAG 1.4.3 AA (4.5:1 normal, 3:1 large), with AAA status also reported. Composites semi-transparent backgrounds via Porter-Duff. Flags gradients, background images, animations, and z-index stacking as caveats. Cannot-calculate cases listed for manual review.

  • WCAG 2.4.7 Focus Visible. Walks every accessible stylesheet and inventories every :focus / :focus-visible / :focus-within rule. Classifies each as suppress-only (outline removed with no replacement — fail), suppress-with-replacement (manual contrast review), replacement-only (safe), or no-style-change. Also flags universal outline:0 resets.

  • WCAG 1.3.1, 1.3.5, 3.3.1, 3.3.2, 4.1.2. Form controls grouped by <form>. Flags unlabeled controls, placeholder-as-label, wrapping <label> (breaks Dragon), poor label position for magnifier users, missing required indicator, required vs aria-required confusion, broken ID refs, missing autocomplete, and forms relying on native browser bubble validation.

  • Heading tree of every <h1>–<h6> and [role="heading"] element, including hidden and sr-only ones. Flags empty headings, skipped levels, multiple <h1>, role="heading" without aria-level, and aria-level out of the 1–6 range.

  • WCAG 2.4.1, 4.1.2. Inventory of every <iframe> (plus obsolete <frame>/<frameset>) across the top document and all nested frames. Flags missing/empty title, generic frame names, names that just repeat the src URL, duplicate names, tabindex="-1" on visible iframes, and aria-hidden over substantial content.

  • Every image-like element (<img>, <svg role="img">, <input type="image">, <area>) classified as labeled, decorative, missing alt, or suspicious alt (filename-looking, generic single word, redundant "image of …" prefix). Shows raw alt alongside computed accessible name.

  • Page-region landmarks (banner, navigation, main, complementary, contentinfo, region, search, form) from HTML5 sectioning and explicit role= attributes. Flags multiple <main>, role="region" without name, and same-role landmarks needing distinct names. Non-qualifying candidates listed for review.

  • Every <a href> and [role="link"] classified per WCAG 2.4.4 / 2.4.9. Flags empty links, generic phrases ("click here", "read more"), URL-as-text, and cross-link clusters (same text pointing to different destinations, or same destination with different texts).

  • WCAG 1.3.1. Every <ul>, <ol>, <menu>, <dl>, and [role="list"] container plus orphan items. Flags non-<li> children of <ul>/<ol>, orphan <li>, empty lists/items, list-style:none without role="list" (Safari/VoiceOver quirk), role="presentation" on natural lists, and broken <dt>/<dd> pairing.

  • WCAG 1.2.1–1.2.5, 1.4.2, 2.2.2. Inventories <video>, <audio>, and third-party player iframes (YouTube, Vimeo, Wistia, etc.). Flags missing controls, autoplay (and autoplay+loop), missing captions, captions present without default, missing transcript nearby (heuristic), and embed URLs with autoplay parameters.

  • WCAG 1.4.11 / 2.4.7. Programmatically focuses each interactive element, reads computed focus styles, measures focus-indicator-vs-background contrast, then restores focus and scroll. Rule families: Input / Button / Link / Tabindex / Handler. Statuses: pass, fail (< 3:1), transparent-outline, gradient/image background (cannot verify), no-focus-indicator.

  • WCAG 1.4.4 Resize Text and 1.4.10 Reflow. Page-level: viewport meta validity (user-scalable, maximum-scale), overflow:hidden on <html>/<body>, current horizontal overflow. Per-element: min-width > 320px, fixed pixel width > 400px without responsive max-width, font-size < 12px. Real reflow testing still requires resize to 320×256.

  • WCAG 2.4.1 Bypass Blocks. Inspects the first 10 focusable elements; flags same-page anchors matching /skip|jump|to main/i or pointing at <main>/<h1>/landmarks. Validates each candidate's target exists, is a main-content anchor, and is focusable (non-interactive targets need tabindex="-1"). Page-level flags: no-skip-link, skip-link-not-first.

  • WCAG 2.4.3 Focus Order. Inventories every focusable element with both its DOM position and the order the browser will actually traverse. Flags positive tabindex (anti-pattern), duplicate positive values, tabindex on non-interactive elements without a role, invalid values, and natively-interactive elements with tabindex="-1". Sort by DOM or focus order.

  • WCAG 1.3.1. Every <table> classified data / layout-declared / ambiguous. Flags data tables without caption/aria-label or without <th>, complex headers missing scope=, spanned cells lacking explicit headers= wiring, multiple captions, obsolete summary= attribute, nested tables, and "looks-like-data-but-no-<th>" (bold first row used as visual headers).

  • WCAG 2.5.8 (AA, 24×24 CSS px) and 2.5.5 (AAA, 44×44 CSS px). Measures every interactive element's bounding rect. Classifies pass, fail-AA, fail-AA-spacing-ok (undersized but spacing exception applies), fail-AAA-only, or inline-link (in-flow-text exception). Hidden, disabled, and zero-size elements are excluded per spec.

  • WCAG 2.4.2 (Page Titled), 3.1.1 (Language of Page), 3.1.2 (Language of Parts). Page title presence and quality, full inventory of every lang and xml:lang attribute with BCP 47 validation, hreflang validation on links, URL-language-hint detection on foreign links, and the "lang= embedded in alt/aria-label" smell check.
