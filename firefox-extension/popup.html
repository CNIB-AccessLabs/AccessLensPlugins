Run a check on this page:

  • Reports the computed accessible name of every interactive element.

  • Heading tree: levels, numbers, skipped levels, empty headings, multiple h1, role="heading" issues.

  • Page-region landmarks (banner, nav, main, complementary, contentinfo, region, search, form) with names. Flags multiple main, unnamed regions, and same-role landmarks needing distinct names.

  • All image-like elements (img, svg, role="img", input type="image", area) classified as labeled, decorative, missing alt, or suspicious alt (filename-looking, generic single word, redundant "image of …" prefix).

  • Link text quality per WCAG 2.4.4 / 2.4.9. Flags empty links, generic phrases ("click here", "read more"), URL-as-text, and cross-link clusters where the same text points to different destinations or vice versa.

  • Validates every role and aria-* attribute on the page. Flags unknown role values, abstract roles, unknown aria-* names (with "did you mean" suggestions), broken aria-labelledby/describedby/controls/owns ID refs, invalid boolean values, missing required attributes per role, and role="presentation"/"none" on focusable elements.

  • Text colour vs background contrast against WCAG 1.4.3 AA (4.5:1 normal, 3:1 large). Composites semi-transparent backgrounds, detects gradients/images/animations/overflow/z-index stacking and surfaces them as caveats. Cannot-calculate cases flagged for manual review. Limitations vs auto_a11y are listed in the panel.

  • WCAG 2.4.2 (Page Titled), 3.1.1 (Language of Page), 3.1.2 (Language of Parts). Page title presence/quality, full inventory of every lang and xml:lang attribute with BCP 47 structural + case-canonical + value validation (region affects screen-reader dialect), lang/xml:lang cross-element mismatch detection, hreflang validation on links, URL-language-hint detection on foreign links, and the "lang= embedded in alt/aria-label" smell check.

  • WCAG 2.4.3 (Focus Order). Full inventory of focusable elements with both their DOM position and the order the browser will actually traverse. Flags positive tabindex (anti-pattern), duplicate positive values, tabindex on non-interactive elements without an interactive role, invalid values, and natively-interactive elements deliberately removed via tabindex="-1". Sort by DOM order or focus order to see where they diverge.

  • WCAG 1.3.1, 1.3.5, 3.3.1, 3.3.2, 4.1.2. Inventory of every form control grouped by <form>. Flags unlabeled controls, placeholder-as-label, wrapping <label>…<input></label> (breaks Dragon NaturallySpeaking), labels not left-aligned above their input (hard to find at high magnification), multiple controls on the same horizontal line (also a magnifier problem), missing visible required indicator, required attribute used where aria-required is correct, broken aria-describedby/labelledby ID refs, missing or invalid autocomplete on personal-data fields, radio/checkbox groups not wrapped in fieldset, empty legends, image inputs without alt, aria-disabled on focusable controls, forms with no submit, and forms relying on native browser bubble validation (bubbles auto-dismiss and are unusable at high zoom).

  • WCAG 1.3.1. Every <table> classified data / layout-declared / ambiguous. Data tables flagged when they lack a caption/aria-label, have no <th>, mix complex headers without scope=, or have spanned cells without explicit headers= wiring. Layout tables flagged when they carry data signals (caption, summary=, aria-label) or when role="presentation"/"none" contradicts the data semantics on the same table. Also flags multiple captions, caption not as the first child, obsolete summary= attribute, nested tables, invalid scope values, empty <th>, broken headers= ID refs, and "looks-like-data-but-no-<th>" (first row visually styled as a header).

  • WCAG 2.4.1, 4.1.2. Inventory of every <iframe> (plus deprecated <frame>/<frameset>) across the top document and all nested frames. Flags missing/empty title, generic frame names ("iframe", "frame", "embedded content", "frame N"), names that just repeat the src URL/hostname/filename, duplicate names across frames, tabindex="-1" on visible iframes (removes the embedded doc from tab order), positive tabindex, aria-hidden="true" on substantial-sized iframes with content (focusable-but-hidden trap), tiny/off-screen iframes without aria-hidden (analytics pixels), role="presentation"/"none" on iframes with content, and empty iframes (no src, no srcdoc). Same-origin vs cross-origin marked per row.

  • WCAG 2.1.1, 4.1.2, 3.3.1/3.3.2. Inventory of every <button>, every <a>, every interactive-role element, and every non-interactive element with inline interaction handlers. Flags <button> inside <form> without explicit type= (defaults to submit), type="reset" (rarely what the user wants), <a href="#"> / href="javascript:" used as buttons, <a> without href, <a disabled> (no effect on links), <a href> with onclick that preventDefault's, <div>/<span> with onclick and no role/tabindex (keyboard-unreachable), interactive ARIA role on non-focusable host with no tabindex, interactive role + onclick but no inline onkey* (heuristic), aria-pressed on non-button, aria-expanded on roles that don't support it, and aria-haspopup without aria-expanded (popup state not exposed). Note: handlers wired via addEventListener can't be statically inspected.

  • WCAG 1.3.1. Inventory of every <ul>, <ol>, <menu>, <dl>, and every [role="list"] container, plus orphan <li>/<dt>/<dd>/[role="listitem"]. Flags non-<li> direct children of <ul>/<ol>, orphan <li>, empty lists, empty list items, <ul>/<ol> with list-style:none and no role="list" (Safari/VoiceOver quirk that strips list semantics), role="presentation"/"none" on natural lists (intentional strip; flagged informationally), [role="list"] without role="listitem" children, orphan role="listitem", <dl> with bad direct children, and broken dt/dd pairing (dt with no dd, dd with no dt).
